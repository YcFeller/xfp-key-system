# 激活密码查询接口安全系统

## 概述

本安全系统为激活密码查询接口（`api.php`）提供了完善的安全防护和日志记录功能，包括：

- 🛡️ **多层安全防护**：IP黑名单、频率限制、自动封禁
- 📊 **完整日志记录**：API访问日志、系统日志、安全事件日志
- 🎯 **智能监控**：实时监控异常行为，自动响应安全威胁
- 🔧 **可视化管理**：Web管理界面，方便查看和管理安全状态

## 系统架构

### 核心组件

1. **SecurityManager.php** - 安全管理核心类
   - IP黑名单管理
   - 请求频率限制
   - 失败尝试记录
   - 自动封禁机制

2. **LogManager.php** - 日志管理核心类
   - 系统日志记录
   - API访问日志
   - 日志轮转和清理
   - 多种日志级别支持

3. **SecurityConfig.php** - 安全配置管理
   - 集中化配置管理
   - 频率限制配置
   - 敏感数据脱敏配置

4. **security_dashboard.php** - Web管理界面
   - 实时安全统计
   - 日志查看和分析
   - IP黑名单管理
   - 系统维护功能

## 安全功能详解

### 频率限制

系统实现了多层级的频率限制：

- **每分钟限制**：5次请求
- **每小时限制**：30次请求  
- **每日限制**：50次请求

### 自动封禁机制

当检测到以下异常行为时，系统会自动封禁IP：

- 1小时内失败尝试超过10次
- 尝试访问不存在的订单
- 尝试访问他人订单
- 使用无效的请求方法
- 提供无效的参数格式

### 安全检查项

1. **用户身份验证**：确保用户已登录
2. **IP黑名单检查**：阻止已封禁的IP访问
3. **请求频率检查**：防止暴力攻击
4. **参数格式验证**：验证订单号和设备码格式
5. **订单所有权验证**：确保用户只能查询自己的订单
6. **验证码校验**：防止自动化攻击

## 日志系统

### 日志类型

1. **系统日志**（system_logs表）
   - 记录系统运行状态
   - 安全事件记录
   - 错误和警告信息

2. **API访问日志**（api_access_logs表）
   - 记录所有API请求
   - 包含请求参数（脱敏）
   - 响应时间和状态码

### 日志级别

- **INFO**：正常操作记录
- **WARNING**：警告信息，需要关注
- **ERROR**：错误信息，需要处理

### 敏感数据脱敏

系统自动对敏感信息进行脱敏处理：

- 订单号：显示前8位，其余用*替代
- 设备码：显示前6位，其余用*替代
- 邮箱：显示前3位，@后保留

## 配置参数说明

### SecurityConfig.php 主要配置

```php
// 频率限制配置
const RATE_LIMITS = [
    'activation_api' => [
        'minute' => 5,      // 每分钟最多5次
        'hour' => 30,       // 每小时最多30次
        'day' => 50         // 每天最多50次
    ]
];

// 自动封禁配置
const AUTO_BAN_CONFIG = [
    'activation_api' => [
        'max_failures' => 10,           // 最大失败次数
        'time_window' => 3600,          // 时间窗口（秒）
        'ban_duration' => 7200          // 封禁时长（秒）
    ]
];

// 安全功能开关
const SECURITY_CONFIG = [
    'enable_rate_limiting' => true,     // 启用频率限制
    'enable_auto_ban' => true,          // 启用自动封禁
    'enable_order_ownership' => true,   // 启用订单所有权检查
    'max_daily_queries' => 50,          // 每日最大查询次数
    'sensitive_data_mask' => true       // 启用敏感数据脱敏
];
```

## 性能优化建议

### 数据库优化

1. **定期清理日志**
   ```sql
   -- 清理30天前的日志
   DELETE FROM system_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);
   DELETE FROM api_access_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);
   ```

2. **索引优化**
   - 确保时间字段有索引
   - IP地址字段建立索引
   - 用户ID字段建立索引

3. **分区表**（可选）
   ```sql
   -- 对大表进行分区，按月分区
   ALTER TABLE system_logs PARTITION BY RANGE (YEAR(created_at)*100 + MONTH(created_at));
   ```

### 缓存优化

可以考虑使用Redis缓存频繁查询的数据：

- IP黑名单缓存
- 频率限制计数器
- 用户权限信息

   - 异常访问模式

2. **性能指标**

### 告警设置建议

- 1小时内失败尝试超过100次
- 单个IP 10分钟内请求超过50次
- 系统错误日志增长异常
- 数据库连接异常

## 故障排除

### 常见问题

1. **日志文件无法写入**
   - 检查目录权限
   - 确认磁盘空间充足
   - 检查PHP错误日志

2. **数据库连接失败**
   - 检查数据库配置
   - 确认数据表已创建
   - 检查数据库用户权限

3. **频率限制不生效**
   - 检查SecurityConfig配置
   - 确认数据表结构正确
   - 查看系统日志错误信息

### 调试模式

在项目根目录 `.env` 中启用调试日志（数据库连接错误会写入 PHP 错误日志）：

```env
APP_DEBUG=true
```
