# XFP邮件系统使用说明

## 概述
本系统集成了基于PHPMailer的邮件发送功能，支持SMTP邮件发送，主要用于密码重置、邮箱验证等安全验证场景。

## 功能特性

### 1. 通用邮件发送类 (`app/mail_sender.php`)
- 支持163邮箱SMTP配置
- 可扩展支持其他邮箱服务商
- 内置HTML邮件模板
- 支持多种验证码类型

### 2. 密码修改功能
- 邮箱验证码验证
- 安全的密码重置流程
- 验证码有效期控制
- 防重复发送机制

### 3. 数据库表结构
需要创建 `verification_codes` 表来存储验证码信息。

## 安装步骤

### 1. 创建数据库表
执行以下SQL语句创建验证码表：

```sql
CREATE TABLE `verification_codes` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `user_id` int(11) NOT NULL,
  `code` varchar(10) NOT NULL,
  `type` varchar(50) NOT NULL DEFAULT 'password_reset',
  `expires_at` datetime NOT NULL,
  `used` tinyint(1) NOT NULL DEFAULT 0,
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `user_id` (`user_id`),
  KEY `type` (`type`),
  KEY `expires_at` (`expires_at`),
  CONSTRAINT `verification_codes_ibfk_1` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2. 配置 SMTP
在项目根目录复制 `.env.example` 为 `.env`，填写以下变量（勿提交 `.env`）：

```env
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_USERNAME=your_email@163.com
SMTP_PASSWORD=your_smtp_authorization_code
MAIL_FROM_ADDRESS=your_email@163.com
MAIL_FROM_NAME=XFP密钥获取系统
```

163 邮箱需使用 SMTP 授权码，而非登录密码。

### 3. 测试邮件功能
在本地编写短脚本调用 `MailSender` 发送测试邮件，或从已集成发信的页面（如密码重置流程）验证。

## 使用方法

### 1. 发送验证码
```php
require_once 'mail_sender.php';

$mailSender = new MailSender();
$code = MailSender::generateVerificationCode(6);
$success = $mailSender->sendVerificationCode($email, $username, $code, 'password_reset');
```

### 2. 验证验证码
```php
// 检查验证码是否有效
$stmt = $conn->prepare("SELECT id FROM verification_codes WHERE user_id = ? AND code = ? AND type = 'password_reset' AND expires_at > NOW() AND used = 0");
$stmt->bind_param('is', $userId, $verificationCode);
$stmt->execute();
$result = $stmt->get_result();
$codeRecord = $result->fetch_assoc();

if ($codeRecord) {
    // 验证码有效，执行相应操作
    // 记得标记验证码为已使用
    $stmt = $conn->prepare("UPDATE verification_codes SET used = 1 WHERE id = ?");
    $stmt->bind_param('i', $codeRecord['id']);
    $stmt->execute();
}
```

## API接口

### 1. 发送验证码
- **URL**: `app/password_reset_api.php`
- **方法**: POST
- **参数**: `action=send_code`
- **返回**: JSON格式的成功/失败信息

### 2. 验证并重置密码
- **URL**: `app/password_reset_api.php`
- **方法**: POST
- **参数**: 
  - `action=verify_and_reset`
  - `verification_code`: 验证码
  - `new_password`: 新密码
  - `confirm_password`: 确认密码
- **返回**: JSON格式的成功/失败信息

## 安全特性

### 1. 验证码安全
- 验证码有效期：10分钟
- 防重复发送：60秒冷却时间
- 一次性使用：验证后立即失效
- 数字验证码：6位纯数字

### 2. 密码安全
- 密码长度验证：最少6位
- 密码确认验证：两次输入必须一致
- 密码哈希存储：使用 `password_hash()` 加密

### 3. 用户验证
- 登录状态检查
- 用户权限验证
- 邮箱所有权验证

- 响应式设计

- 安全提醒信息
- 验证码突出显示

## 扩展功能

### 1. 支持其他验证码类型
可以通过修改 `type` 参数支持不同类型的验证码：
- `password_reset`: 密码重置
- `email_verify`: 邮箱验证
- `account_security`: 账户安全验证

### 2. 支持其他邮箱服务商
可以通过修改SMTP配置支持其他邮箱服务商：
- QQ邮箱：smtp.qq.com:587
- Gmail：smtp.gmail.com:587
- 企业邮箱：根据服务商配置

## 故障排除

### 1. 邮件发送失败
- 检查SMTP配置是否正确
- 确认邮箱密码是否为SMTP专用密码
- 检查服务器防火墙设置
- 查看PHP错误日志

### 2. 验证码不生效
- 检查数据库表是否创建成功
- 确认验证码是否过期
- 检查验证码是否已被使用
- 验证用户ID是否正确

### 3. 页面显示错误
- 检查PHP版本兼容性
- 确认PHPMailer文件路径正确
- 检查文件权限设置
- 查看浏览器控制台错误

## 注意事项

1. **SMTP密码**: 请使用邮箱的SMTP专用密码，不是登录密码
2. **服务器配置**: 确保服务器支持SMTP连接
3. **邮件频率**: 避免频繁发送邮件，防止被邮箱服务商限制
4. **数据安全**: 定期清理过期的验证码记录
5. **备份**: 定期备份验证码表数据
